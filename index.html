Instructions

Create a Broadcom Bluecoat ProxySG CPL policy by entering the following:

  • CI Number: A unique identifier for the policy (e.g., 12345). Use alphanumeric characters and underscores only.
  • Purpose: Optional description of the taskâ€™s purpose (e.g., Allow IT team access for project X). Avoid semicolons.
  • Comment: Optional description for the policy (e.g., Policy for IT team access). Avoid semicolons.
  • Domain: Select the domain for groups and users in Source (required if using groups or users).
  • Source: Comma-separated groups, users (e.g., group_IT/james), IP addresses, or subnets (e.g., group_IT,group_HR,group_IT/james,1.1.1.1,192.168.1.0/24). Can be empty if Destination is provided.
  • Destination: Comma-separated IPs, subnets, or URLs (e.g., 8.8.8.8,10.0.0.0/16,google.com). Can be empty if Source is provided.
  • Action: Select either "Allow" or "Deny" to define the policy action.
  • Forwarding Layer: Choose if a forwarding layer is needed. If yes, select an upstream proxy host and the same conditions will apply in a <Forward> rule.
  • Click "Generate CPL" to create the policy, which will appear below.
